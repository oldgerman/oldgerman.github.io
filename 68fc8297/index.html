C++11 中值得关注的几大变化（详解）

C98 的知能指针是 auto_ptr， 在C++ 11中被废弃了。C11 引入了两个指针类： shared_ptr 和 unique_ptr。 shared_ptr只是单纯的引用计数指针，unique_ptr 是用来取代auto_ptr。 unique_ptr 提供 auto_ptr 大部份特性，唯一的例外是 auto_ptr 的不安全、隐性的左值搬移。不像 auto_ptr，unique_ptr 可以存放在 C++0x 提出的那些能察觉搬移动作的容器之中。

为什么要这么干？大家可以看看《More Effective C++》中对 auto_ptr的讨论。

rvalue-reference 右值引用（C++11）

在C++11新标准中引入了一种新的引用类型——“右值引用”，主要用在类的内部。技术上当我们说引用，不特指的情况下都是指“左值引用”。

首先需要区分左值(lvalue)和右值(rvalue)。 左值和右值是从C里继承过来的，在C里很容易区分：左值可以放在一个赋值语句的左手边，而右值不可以。但在C里，情况会复杂得多：在C，一个左值表达式会产生一个对象或函数，但一些左值比如const对象，不能被再赋值，不能成为左值；比如一些表达式产生对象，但却返回右值。简单地概括，当我们使用一个对象作为右值的时候，我们使用的是对象的值（内容），而当我们使用一个对象作为左值的时候，我们使用的是它的身份（也就是内存中的位置）。左值是持久的，右值是短暂的，变量都是左值。

右值引用指的是一个引用必须绑定到右值上，采用&&标识符表示。右值引用又一个重要的性质——只能绑定到一个将要销毁的对象，这样我们就可以自由地将右值引用地资源std::move到另一个对象中。

std::move是C++11中的新标准库函数，因为不能将一个右值引用绑定到左值上，可以用这个函数来获得绑定到左值上的右值引用。

move函数调用是告诉编译器：我们这儿又一个左值，但我希望像一个右值一样处理它，也就意味着，除了对rr1赋值或销毁它之外，我们将不再使用它。

关于vptr和vtbl（virtual pointer虚指针、virtual table虚表）

只要你的class内有一个（或一万个）虚函数，那么class的实例化对象的内存块就会多一个指针，当我们去测它的大小会发现比其携带的数据大小多4字节

"为什么A有虚函数B一定要有虚函数？"
从什么是继承回答：

从内存的角度想，继承就是把数据全部继承下来，这个话没错，但还会把函数继承下来，函数占内存多大？不可解。所谓函数继承的是它的调用权,不是继承它的内存大小，所以B中必定要有虚函数

virtual table 虚表

virtual table里面的地址解引用都是 指向虚函数的函数指针类型

对于class A，内部有两个virtual function，那么其虚表就有两个虚函数指针指向两个虚函数

对于class B, 内部有两个virtual function，那么其虚表就有两个虚函数指针指向两个虚函数，但是B重写了对从A继承的vfunc1()，即在动态解析时vfunc1()会根据被调对象的class不同而使用class内部不同的实现，所以其中一个函数指针指向B重写的vfunc1()函数新的地址，而不是A的vfunc1()函数的地址，B没有改写vfunc2()，所以另一个函数指针指向原来A的vfunc2()函数的地址

对于class c，也有自己的vfunc1()，与B类似

理解 C++ 虚函数表

“可以看到，Derived 类中重写的 vPrint() 方法已经被执行。这就说明虚函数在执行时是一个动态的过程，并不是在编译时就确定下来要执行哪一个函数，而是运行时从虚表查到真正要执行的函数的地址，然后再将 this 指针传入执行。”

虚函数的Polymorphism（多态）用法

对于C++编译器编译一个Diver函数，要判断它是静态绑定还是动态绑定，静态绑定编译成汇编形式的call xxx，xxx的值写死到某个地址，如果符合以下3个条件就会Dynamic Binding动态绑定：

  • 使用指针
  • 指针要up-cast（向上转型），例如指针声明为指向Base类的，但使其指向new的Derived对象（是不是每个实例化对象的this指针默认声明为指向基类的指针？然后使其指向本身的实例化对象的类？）
  • 调用的是虚函数

这个又称virtual-mechanism（虚机制），即动态绑定，与传统的C相比显著减少if-else的使用

虚函数的Template Method（模板方法）用法

该方法是子类对象调用父类的普通函数，而这个函数又调用了虚函数，根据被调函数的对象的this指针判断是子类调用了虚函数，从而将虚函数解析为子类的实现，与借助指向子类的父类指针的解析相反

C++中，成员函数一定有一个隐藏的参数：this

上述mian()里的myDoc.OnFileOpen();写成C的形式是CDocument::OnFileOpen(&myDoc); // this就是&myDoc

上述CDocument::OnFileOpen()里的Serialize()显式this写法就是：this->Serialize();，当传入CMyDoc的this写成C形式就是：(*(this->vptr)[n])(this);，因为这么整符合Dynamic Binding的三个条件，所以成功解析为子类的虚函数实现
